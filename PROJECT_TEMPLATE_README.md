## 📁 生成的项目结构

```

├── pom.xml                          # Maven 项目配置
├── .gitignore                       # Git 忽略文件
├── src/
│   └── main/
│       ├── java/
│       │   ├── {groupId}/{artifactId}/
│       │   │   ├── event/           # 事件处理
│       │   │   ├── trigger/         # 触发器
│       │   │   ├── service/         # 服务层
│       │   │   ├── futuretask/      # 异步任务
│       │   │   ├── util/            # 工具类
│       │   │   └── html/            # HTML 资源
│       │   └── scriptTrigger.xml    # 脚本触发器配置
│       └── resources/
│           └── oauthConfig.properties  # OAuth 配置
```

## 🚀 如何使用

### 1. 创建新项目

1. 打开 IntelliJ IDEA
2. File → New → Project
3. 选择 **"SalesEasy Project"**
4. 填写配置信息:
   - **GroupId**: `other.xsy`
   - **ArtifactId**: `jinpan`
   - **OAuth 配置**: 填写或留空

5. 点击 **Create**

### 2. 项目创建后

IntelliJ IDEA 会自动:
- ✅ 识别为 Maven 项目
- ✅ 导入 Maven 依赖
- ✅ 配置源代码目录
- ✅ 设置编译器版本 (Java 1.8)

### 3. 开始开发

项目创建后,您可以:
- 在 `event/` 目录下创建事件处理类
- 在 `trigger/` 目录下创建触发器类
- 在 `service/` 目录下创建业务逻辑
- 在 `pom.xml` 中添加依赖

## 📝 配置说明

### GroupId 和 ArtifactId

- **GroupId**: 组织或公司的唯一标识,通常使用反向域名
  - 例如: `other.xsy`, `com.example`
  
- **ArtifactId**: 项目的唯一标识
  - 例如: `jinpan`, `my-project`

### OAuth 配置

OAuth 配置是可选的:
- 如果不需要,可以全部留空
- 如果需要,必须填写: UserName, Password, Domain
- 其他字段 (SecurityCode, ClientId, ClientSecret) 可选

## 🔧 自定义配置

### 修改 Java 版本

编辑 `pom.xml`:

```xml
<properties>
    <maven.compiler.source>11</maven.compiler.source>
    <maven.compiler.target>11</maven.compiler.target>
</properties>
```

### 添加依赖

在 `pom.xml` 的 `<dependencies>` 标签中添加:

```xml
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-web</artifactId>
    <version>2.7.0</version>
</dependency>
```

## 🐛 故障排除

### 问题: Maven 依赖无法下载

**解决方案:**
1. 检查网络连接
2. 配置 Maven 镜像 (settings.xml)
3. 使用 `mvn clean install -U` 强制更新

### 问题: 项目未被识别为 Maven 项目

**解决方案:**
1. 右键项目 → Add Framework Support → Maven
2. 或者: File → Project Structure → Modules → Import Module
