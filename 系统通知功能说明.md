# 系统通知功能说明

## ✅ 新增功能

### 系统气泡通知

当有新通知到达时，会在IntelliJ IDEA右下角弹出系统通知气泡，类似于您截图中看到的效果。

## 功能特点

### 1. 自动检测新通知

- 每30秒自动刷新通知
- 比较刷新前后的通知数量
- 如果有新通知，自动弹出系统通知

### 2. 通知内容

**标题**: `收到 X 条新通知`
- X 是新增的通知数量

**内容**: 显示最新一条通知的标题
- 例如："新的BUG分配"、"BUG已提交复测"等

### 3. 通知样式

- **类型**: BALLOON（气泡通知）
- **位置**: IntelliJ IDEA 右下角
- **持续时间**: 自动消失（可点击查看详情）
- **日志**: 自动记录到事件日志

## 工作流程

```
自动刷新（每30秒）
  ↓
获取最新通知列表
  ↓
比较通知数量
  ↓
发现新通知？
  ↓ 是
弹出系统通知气泡
  ↓
显示新通知数量和标题
```

## 技术实现

### 1. 注册通知组

在 `plugin.xml` 中注册：
```xml
<notificationGroup id="SalesEasy.Notifications" 
                  displayType="BALLOON" 
                  isLogByDefault="true"/>
```

### 2. 检测新通知

```kotlin
// 记录刷新前的通知数量
val oldNotificationCount = notificationList.size()

// 刷新后比较
val newNotificationCount = notifications.size
if (newNotificationCount > oldNotificationCount) {
    // 有新通知
    val newCount = newNotificationCount - oldNotificationCount
    showSystemNotification(...)
}
```

### 3. 显示系统通知

```kotlin
private fun showSystemNotification(title: String, content: String) {
    val notificationGroup = NotificationGroupManager.getInstance()
        .getNotificationGroup("SalesEasy.Notifications")
    
    val notification = notificationGroup.createNotification(
        title,
        content,
        NotificationType.INFORMATION
    )
    notification.notify(project)
}
```

## 配置选项

### 通知显示类型

在 `plugin.xml` 中可以修改：

| displayType | 说明 |
|-------------|------|
| BALLOON | 气泡通知（默认，推荐） |
| STICKY_BALLOON | 粘性气泡（需要手动关闭） |
| TOOL_WINDOW | 工具窗口通知 |
| NONE | 仅记录日志 |

### 是否记录日志

```xml
isLogByDefault="true"   <!-- 记录到事件日志 -->
isLogByDefault="false"  <!-- 不记录日志 -->
```

## 使用说明

### 自动通知

1. **登录系统**
2. **等待30秒**（或有新通知时）
3. **系统自动弹出通知气泡**
4. **点击气泡可查看详情**

### 手动刷新

1. **点击"刷新"按钮**
2. **不会弹出系统通知**（手动刷新不触发）
3. **直接在通知列表中查看**

## 通知触发条件

### ✅ 会触发系统通知

- 自动刷新（每30秒）
- 检测到新通知
- 通知数量增加

### ❌ 不会触发系统通知

- 手动点击刷新按钮
- 首次登录加载通知
- 通知数量没有变化
- 通知数量减少

## 优化建议

### 1. 避免通知轰炸

当前实现已经考虑了这一点：
- 只在自动刷新时触发
- 手动刷新不触发
- 避免频繁打扰用户

### 2. 通知分组

如果需要按类型分组通知：
```kotlin
when (notification.type) {
    NotificationType.ERROR -> showSystemNotification("错误通知", ...)
    NotificationType.WARNING -> showSystemNotification("警告通知", ...)
    NotificationType.INFO -> showSystemNotification("信息通知", ...)
    NotificationType.SUCCESS -> showSystemNotification("成功通知", ...)
}
```

### 3. 自定义通知图标

可以为不同类型的通知设置不同图标：
```kotlin
notification.setIcon(AllIcons.General.Information)
```

## 常见问题

### Q: 为什么没有弹出通知？

A: 检查以下几点：
1. 是否已登录
2. 是否有新通知（数量是否增加）
3. 是否是自动刷新触发的
4. 通知设置是否被禁用

### Q: 如何禁用系统通知？

A: 修改代码，在 `refreshNotifications` 中注释掉：
```kotlin
// showSystemNotification(...)
```

### Q: 如何修改通知样式？

A: 修改 `plugin.xml` 中的 `displayType`：
```xml
<notificationGroup id="SalesEasy.Notifications" 
                  displayType="STICKY_BALLOON"  <!-- 改为粘性气泡 -->
                  isLogByDefault="true"/>
```

### Q: 通知能否播放声音？

A: 可以，但需要额外配置：
```kotlin
notification.setImportant(true)  // 标记为重要通知
```
