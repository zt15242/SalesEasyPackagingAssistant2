# 通知同步功能使用说明

## 功能概述

现在插件已经集成了真实的API服务，可以从后端同步通知数据。

## 使用步骤

### 1. 配置API域名

确保 `ApiConfig.kt` 中的域名配置正确：

```kotlin
var BASE_URL: String = "http://114.66.56.200:6002"
```

### 2. 选择服务模式

在使用前需要选择使用真实API还是模拟数据。

**使用真实API（推荐）**:
```kotlin
ServiceFactory.setServiceMode(useMock = false)
```

**使用模拟数据（测试）**:
```kotlin
ServiceFactory.setServiceMode(useMock = true)
```

### 3. 登录系统

1. 打开 IntelliJ IDEA
2. 找到 "SalesEasy Assistant" 工具窗口
3. 在"用户登录"区域输入用户名和密码
4. 点击"登录"按钮
5. 登录成功后会自动加载通知

### 4. 查看通知

登录成功后，通知中心会自动显示从后端获取的通知列表，包括：
- 新的BUG分配
- BUG已提交复测
- BUG新评论
- 等等...

### 5. 刷新通知

点击"刷新"按钮可以重新从后端获取最新的通知。

### 6. 清空通知

点击"清空"按钮可以清空本地显示的通知列表（不会删除服务器上的通知）。

## 功能说明

### 登录功能

- ✅ 使用真实的后端API进行认证
- ✅ 自动获取用户信息
- ✅ 自动管理JWT Token
- ✅ 登录成功后自动加载通知
- ✅ 支持登出功能

### 通知功能

- ✅ 从后端API获取通知列表
- ✅ 支持分页加载（默认50条）
- ✅ 显示通知标题、内容和时间
- ✅ 根据通知类型显示不同图标
- ✅ 支持手动刷新
- ✅ 登出时自动清空通知

### 通知类型

系统支持以下通知类型：

| 类型 | 图标 | 说明 |
|------|------|------|
| INFO | ℹ️ | 普通信息 |
| SUCCESS | ✅ | 成功消息 |
| WARNING | ⚠️ | 警告信息 |
| ERROR | ❌ | 错误信息 |

## 工作流程

```
1. 用户输入用户名密码
   ↓
2. 点击登录按钮
   ↓
3. 调用 POST /api/auth/login
   ↓
4. 获取 access_token
   ↓
5. 调用 GET /api/auth/me
   ↓
6. 获取用户信息
   ↓
7. 自动调用 GET /api/notifications/
   ↓
8. 显示通知列表
```

## 刷新流程

```
1. 点击刷新按钮
   ↓
2. 检查登录状态
   ↓
3. 调用 GET /api/notifications/?skip=0&limit=50
   ↓
4. 清空现有通知列表
   ↓
5. 添加新获取的通知
   ↓
6. 显示刷新成功提示
```

## 错误处理

### 未登录

如果未登录就点击刷新，会显示警告：
```
⚠️ 未登录
请先登录后再刷新通知
```

### 网络错误

如果网络请求失败，会显示错误信息：
```
❌ 刷新失败
无法获取通知: [错误详情]
```

### 登录失败

如果用户名或密码错误，会显示：
```
❌ 登录失败
登录失败: [错误详情]
```

## 配置说明

### 修改通知加载数量

在 `SalesEasyToolWindowPanel.kt` 的 `refreshNotifications()` 方法中修改：

```kotlin
val notifications = notificationService.getNotifications(
    page = 1, 
    pageSize = 50  // 修改这里的数量
)
```

### 启用自动刷新

可以添加定时器自动刷新通知：

```kotlin
// 每30秒自动刷新一次
Timer(30000, ActionListener {
    if (isLoggedIn) {
        refreshNotifications()
    }
}).start()
```

### 切换服务模式

**方法1**: 在代码中设置
```kotlin
// 在 SalesEasyToolWindowPanel 初始化时
init {
    // 使用真实API
    ServiceFactory.setServiceMode(useMock = false)
    setupUI()
}
```

**方法2**: 通过系统属性
```kotlin
val useMock = System.getProperty("use.mock.services", "false").toBoolean()
ServiceFactory.setServiceMode(useMock = useMock)
```

然后在运行配置中添加 VM 选项：
```
-Duse.mock.services=false
```

## 常见问题

### Q: 为什么登录后没有显示通知？

A: 可能的原因：
1. 后端没有通知数据
2. API域名配置错误
3. 网络连接问题
4. Token认证失败

解决方法：
- 检查 `ApiConfig.BASE_URL` 配置
- 查看控制台错误日志
- 手动点击"刷新"按钮重试

### Q: 如何查看详细的错误信息？

A: 
1. 打开 IntelliJ IDEA 的控制台
2. 查看异常堆栈信息
3. 检查 `HttpClient` 返回的错误消息

### Q: 通知显示的时间不对？

A: 当前实现中 `parseTimestamp()` 方法返回的是当前时间，需要改进：

```kotlin
private fun parseTimestamp(timestamp: String): Long {
    return try {
        val formatter = java.time.format.DateTimeFormatter.ISO_DATE_TIME
        val dateTime = java.time.LocalDateTime.parse(timestamp, formatter)
        dateTime.atZone(java.time.ZoneId.systemDefault()).toInstant().toEpochMilli()
    } catch (e: Exception) {
        System.currentTimeMillis()
    }
}
```

### Q: 如何添加更多通知操作？

A: 可以在通知列表中添加右键菜单：

```kotlin
jList.addMouseListener(object : MouseAdapter() {
    override fun mouseClicked(e: MouseEvent) {
        if (SwingUtilities.isRightMouseButton(e)) {
            val index = jList.locationToIndex(e.point)
            if (index >= 0) {
                jList.selectedIndex = index
                showNotificationMenu(e.x, e.y)
            }
        }
    }
})
```

## 性能优化建议

1. **分页加载**: 当通知很多时，使用分页加载
2. **缓存**: 缓存已加载的通知，减少API调用
3. **增量更新**: 只获取新的通知，而不是每次都全量加载
4. **后台刷新**: 使用后台线程定期刷新，不阻塞UI
