# 快速配置指南 - 修改API域名

## 当前配置

- **默认域名**: `http://114.66.56.200:6002`
- **配置文件**: `src/main/kotlin/top/javaxiaowu/saleseasypackagingassistant/config/ApiConfig.kt`

## 修改方法

### 方法1: 修改配置文件（推荐）

1. 打开文件: `src/main/kotlin/top/javaxiaowu/saleseasypackagingassistant/config/ApiConfig.kt`

2. 找到第12行左右的代码:
```kotlin
var BASE_URL: String = "http://114.66.56.200:6002"
```

3. 修改为新的域名:
```kotlin
var BASE_URL: String = "http://your-new-domain.com:port"
```

4. 保存文件并重新编译插件

### 方法2: 代码中动态修改

在代码中任何位置调用:
```kotlin
import top.javaxiaowu.saleseasypackagingassistant.config.ApiConfig

// 修改域名
ApiConfig.setBaseUrl("http://new-domain.com:8080")
```

### 方法3: 支持环境变量（需要扩展）

如果需要通过环境变量配置，可以修改 `ApiConfig.kt`:

```kotlin
var BASE_URL: String = System.getenv("API_BASE_URL") 
    ?: System.getProperty("api.base.url")
    ?: "http://114.66.56.200:6002"
```

然后通过以下方式设置:

**Windows (PowerShell)**:
```powershell
$env:API_BASE_URL = "http://new-domain.com:8080"
```

**Linux/Mac**:
```bash
export API_BASE_URL="http://new-domain.com:8080"
```

**IntelliJ Run Configuration**:
1. Edit Configurations
2. Environment variables: `API_BASE_URL=http://new-domain.com:8080`

## 常见问题

### Q: 修改域名后需要重启吗?
A: 如果在配置文件中修改，需要重新编译插件。如果使用 `setBaseUrl()` 动态修改，立即生效。

### Q: 可以使用HTTPS吗?
A: 可以，直接设置为 `https://` 开头的URL即可。

### Q: 端口号是必须的吗?
A: 如果使用标准端口（HTTP:80, HTTPS:443），可以省略端口号。

### Q: 如何在不同环境使用不同域名?
A: 建议使用环境变量或配置文件，根据环境加载不同的配置。

## 注意事项

1. **URL格式**: 
   - ✅ 正确: `http://domain.com:8080`
   - ✅ 正确: `https://domain.com`
   - ❌ 错误: `http://domain.com:8080/` (末尾不要加斜杠)

2. **协议**:
   - 开发环境可以使用 `http://`
   - 生产环境建议使用 `https://`

3. **端口**:
   - 确保端口号正确
   - 确保防火墙允许访问该端口

4. **网络**:
   - 确保插件运行环境可以访问该域名
   - 检查网络代理设置

## 快速切换域名脚本

可以在插件设置中添加域名选择:

```kotlin
// 预定义的域名配置
object DomainPresets {
    const val LOCAL = "http://localhost:8000"
    const val DEV = "http://114.66.56.200:6002"
    const val TEST = "http://test-server.com:8000"
    const val PROD = "https://api.production.com"
}

// 快速切换
fun switchToDev() {
    ApiConfig.setBaseUrl(DomainPresets.DEV)
}

fun switchToProd() {
    ApiConfig.setBaseUrl(DomainPresets.PROD)
}
```
